# GitHub

GitHub ek cloud-based hosting platform hai jaha tum git repositories ko store, manage aur share kar sakte ho.

GitHub ek website hai jaha pe tum apni local repositories ko store karte ho aur team ke saath us repository ko share karte ho.

GitHub ek server hai jaha hum code push karte hain aur koi aur dusra waha se code pull karta hai:
- Git ek tool hai (local machine par chalta hai).
- GitHub ek service hai (internet pe host hoti hai).


Jaise tumhare paas code hai to tum Git se version control karoge. Aur GitHub par tum use store karke team members ke sath collaborate karoge.

<br>
<br>

### Git vs GitHub

| Aspect           | Git                                                   | GitHub                                                                            |
| ---------------- | ----------------------------------------------------- | --------------------------------------------------------------------------------- |
| Kya hai          | Command-line based distributed version control system | Web-based hosting + collaboration platform for Git repos                          |
| Kahan chalta hai | Tumhare computer/local server                         | Cloud (github.com)                                                                |
| Core kaam        | Code track karna, history maintain karna              | Code ko remote store karna, team ke saath kaam karna, pull request, CI/CD, issues |
| Free/paid        | Free open source                                      | Free (public repos) + Paid (private/org features)                                 |

<br>
<br>

### Workflow of GitHub

We know that github par hum apna local code ko upload karte hain, code ko upload karne ke liye ya fir code ko github se apne local system par copy karne ke liye techniques hoti hain:

**Push**: 

Is technique se local system par bani git repository ko github par push kiya jata hai.

<br>

**Pull**:

Is technique se repository ko github se local system par pull kiya jata hai.


- Repo create karo GitHub par.
- Local machine par code likho aur Git se track karo.
- Code ko ```git push``` karke GitHub repo me bhejo.
- Team members GitHub se code clone/pull karke apne local par kaam kare.
- Naye features branch me develop karo, PR banao, review/merge karo.

<br>
<br>

### GitHub ke Core Concepts

**1 - Repository (Repo)**:
- Ye tumhara project ka container hai.
- Har repo ke andar tumhara code, commits, branches, tags, issues sab hota hai.
- Repo public (open source) ya private (restricted) ho sakta hai.

Example:
```
https://github.com/puneet/my-python-app
```

<br>

**2 - Pull Requests (PR)**:
- Jab tum ek branch (feature) ko dusre branch (main) me merge karna chahte ho to GitHub par PR banate ho.
- PR ek request hota hai ek branch ko dusri branch main merge karne ka.
- PR ke baad merge karte ho → code main me aa jata hai.

<br>

**3 - Issues**:
- Ye GitHub ka bug tracker / task manager hai.
- Tum bug report, feature request ya discussion create kar sakte ho.
- Labels, assignees, milestones laga sakte ho.

<br>

**4 - Wiki**:
- Repo ka built-in documentation area.
- Team yahan documentation maintain kar sakti hai.

<br>

**5 - Actions**:
- GitHub ka CI/CD system.
- Tum ```.github/workflows/ci.yml``` me pipeline define kar sakte ho.
- Jaise jab PR open ho to Python tests run karo, docker image build karo, deploy karo.

<br>

**6 - Security**:
- Dependabot automatic dependency updates karta hai.
- Code scanning se vulnerabilities detect hoti hain.

<br>

**7 - GitHub Pages**:
- Repo se directly static website host kar sakte ho.

<br>
<br>

### How to Push code to github

**Sabse pehle — Push ka basic meaning**:

```git push``` ka matlab hai:
- Tumhare local repository (jo tumhare computer ke .git folder me hai) ke commits ko remote repository (jaise GitHub, GitLab, Bitbucket) par bhejna.
- Push sirf commits ko bhejta hai. Ye tumhare working directory ya staging area ka raw code nahi bhejta, balki tumhare commits (Git objects) ko remote ke refs par update karta hai.

<br>

**Push hone se pehle kya hota hai**:

Local me Git ke 3 zones hote hain:
- Working Directory: jahan tum files edit karte ho.
- Staging Area / Index: jahan tum ```git add``` karke commits prepare karte ho.
- Local Repository (.git): jahan tumhare commits actual store hote hain.

```git push``` tabhi chal sakta hai jab tumhare local repository me commits hain jo remote me nahi hain.

Flow:
- File edit karo.
- Staging area me add karo (```git add file.py```).
- Commit karo (```git commit -m "message"```).
- Push karo (```git push origin branch-name```).

<br>

**Push karne ke liye remote ka concept**:

- Remote repository = ek server (GitHub/GitLab/Bitbucket/self-hosted) jahan tumhara Git repo hai.
- Git me remote repository ka naam default hota hai ```origin``` (but tum kuch bhi rakh sakte ho).
- Remote URLs stored hote hain ```.git/config me```.

Check remotes:
```
git remote -v
```

Output:
```
origin  https://github.com/username/myrepo.git (fetch)
origin  https://github.com/username/myrepo.git (push)
```

Yani tumhara remote origin hai aur uska URL GitHub par hai.

<br>

**git push ka internal process**:

Jab tum git push origin main likhte ho to ye hota hai:
- Negotiation phase:
  - Tumhara Git remote se connect hota hai (HTTP/HTTPS, SSH).
  - Remote se poochta hai: “mere paas yeh yeh commits hain, tere paas kaunse hain?”.
  - Remote reply karta hai apne refs/commits list se.

- Pack phase:
  - Tumhara Git jo commits remote par nahi hain unko packfile me compress karta hai (pack objects).
  - Saath me unke blobs (files content) aur trees (directory structure) bhi include karta hai.
  - Is packfile ko network par bhejta hai.
 
- Update refs phase:
  - Remote repo packfile unpack karta hai, nayi objects apne ```.git/objects``` me store karta hai.
  - Remote branch pointer update karta hai (e.g. ```refs/heads/main``` ko naye commit SHA par).

Agar remote protected hai (branch protection), ya tumhare paas permission nahi hai, ya fast-forward possible nahi hai, to push reject ho sakta hai.

<br>

**Common Push Scenarios**:

- First push of a branch:
```
git push -u origin new-branch
```
```-u``` sets the upstream so next time you can just do ```git push```.

- Push after new commits:
```
git push
```
Sends new commits to remote.
