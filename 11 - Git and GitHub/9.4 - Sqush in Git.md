# Sqush in Git

<br>

### Git Squash kya hai?

Git squash ek process hai jisme tum multiple commits ko combine karke ek single commit bana dete ho.

Iska use commit history ko clean aur readable banane ke liye hota hai.

Soch lo:
- Tum ek feature develop kar rahe ho.
- Tumne 8 chhote-chhote commits kiye (typo fixes, small changes, debugging etc.).
- Tum chahte ho ki jab ye feature ```main``` branch me merge ho to ek hi commit dikhai de (“Add new payment feature”).
- Yaha tum squash karte ho.

Result: Commit history clean, ekdum professional.

<br>
<br>

### Squash kyon use karte hain?

- **History Clean Karne ke liye** – Bade feature ke multiple WIP commits ko combine karke ek meaningful commit ban jata hai.
- **PR (Pull Request) Clean Karne ke liye** – Reviewer ke liye ek hi commit dekhna easy hota hai.
- **Typos aur Junk Commits Hatane ke liye** – “fix typo”, “oops forgot semicolon” type commits ek commit me squash kar do.
- **Git bisect easy hota hai** – Jab commits meaningful hote hain to bug track karna easy hota hai.

<br>
<br>

### Git me Squash karne ke 3 Common Tareeke
- Interactive Rebase se squash karna (most common).
- Merge karte waqt squash karna (--squash flag).
- GitHub/GitLab UI se squash (GUI option).

<br>
<br>

### Interactive Rebase se Squash (Sabse powerful)

Scenario:
- Tumhare branch ```feature-xyz``` me commits kuch aise hain:
```
git log --oneline
```
Output:
```
a1b2c3d Add payment feature (final)
d4e5f6g Add tests for payment
h7i8j9k Fix typo in payment
l0m1n2o Initial commit for payment
```

Tum chahte ho ki last ke 3 commits (Add tests, Fix typo, Initial commit) ek hi commit ban jaye.

**Step 1: Interactive Rebase Start karo**:
```
git rebase -i HEAD~3
```
(HEAD~3 ka matlab last 3 commits ko edit karna.)

**Step 2: Editor khulega**:
```
pick d4e5f6g Add tests for payment
pick h7i8j9k Fix typo in payment
pick l0m1n2o Initial commit for payment
```

Tumko “pick” ko change karke “squash” (ya “s”) karna hota hai un commits par jise upar wale ke saath combine karna hai:
```
pick d4e5f6g Add tests for payment
squash h7i8j9k Fix typo in payment
squash l0m1n2o Initial commit for payment
```

**Step 3: Commit Message Edit karo**:

Next screen me Git tumse commit message puchega:
```
# This is a combination of 3 commits.
# The first commit's message is:
Add tests for payment

# The following commit messages will also be included:
Fix typo in payment
Initial commit for payment
```

Tum apna final message likh do:
```
Add payment feature with tests
```

Save and exit.

**Step 4: Rebase Complete ho gaya**:

Ab ```git log --oneline``` karo:
```
a1b2c3d Add payment feature (final)
x9y8z7a Add payment feature with tests
```

Dekha? Last 3 commits ek commit me squash ho gaye.

<br>
<br>

### Merge karte waqt Squash

Agar tum ek branch ko merge kar rahe ho aur chahte ho ki us branch ke saare commits ek hi commit me ho jaye, to tum ```--squash``` flag use kar sakte ho.

Example:
```
git checkout main
git merge --squash feature-xyz
git commit -m "Add payment feature"
```

Ye ```feature-xyz``` ke saare commits ko combine karke ek commit banata hai. Git history clean rehti hai.
